# Lumina Core

Lumina Core is the foundational data and service layer of the Lumina ecosystem.

It connects library data, metadata, embeddings, semantic services, APIs and application components into a coherent knowledge infrastructure.

Lumina Core should be visualised as stable, structured and central.

It is not the most visible part of Lumina, but it enables everything else.

---

## Purpose

Lumina Core provides the shared foundation for Lumina services.

It supports:

- structured data management
- metadata storage and access
- semantic data connections
- API-based service access
- configuration and orchestration support
- shared knowledge objects
- reusable service layers
- integration between Lumina components

Lumina Core should feel like the trusted backbone of the ecosystem.

It is the place where data, services and meaning come together.

---

## Core Message

Lumina Core turns distributed library data and semantic services into a reliable, reusable and connected foundation.

The key message is:

**Lumina Core provides the trusted infrastructure that makes intelligent library services possible.**

It should communicate:

- stability
- structure
- trust
- reusability
- interoperability
- connected knowledge
- service orientation
- institutional reliability

Lumina Core should not feel experimental or fragile.

It should feel like a dependable foundation.

---

## Role in the Lumina Ecosystem

Lumina Core sits between data, processing and user-facing services.

It connects:

- Lumina Engine
- Lumina Studio
- Lumina Experience
- APIs
- metadata repositories
- semantic services
- search and discovery systems
- enrichment workflows
- administrative configurations

Lumina Core should be visualised as the shared centre or foundation layer of the Lumina ecosystem.

It does not replace the other components.

It enables them.

---

## Visual Story

The visual story of Lumina Core should be about connection, structure and trust.

Possible story:

Library data and semantic knowledge objects flow into a central, well-structured core. From there, reusable services and APIs make this knowledge available to Lumina Engine, Lumina Studio and Lumina Experience.

The visual should show:

- inputs becoming structured
- services becoming reusable
- components being connected
- knowledge being stabilised
- APIs acting as clean access points
- data and metadata forming a reliable foundation

The viewer should understand that Lumina Core is not just storage.

It is the structured service layer that holds the Lumina ecosystem together.

## Preferred Composition

Lumina Core works well in architecture-style compositions.

Preferred layouts:

### Layered Architecture

A stable foundation layer at the bottom or centre, with Lumina components above or around it.

Good for explaining that Core supports the ecosystem.

### Central Hub

Lumina Core appears as a central structured hub connected to Engine, Studio, Experience and external services.

Good for explaining interoperability and shared services.

### Service Platform

Lumina Core appears as a set of modular service blocks with clean API connections.

Good for explaining APIs and reusability.

### Data-to-Service Flow

Library data and metadata enter from one side, become structured in Lumina Core, and leave as reusable services.

Good for explaining transformation from data to usable services.

## Colour Emphasis

Use:

- ETH Blue for the central Core layer or main structural elements
- Black for labels, icons and structural contrast
- White for background and calm space
- ETH Petrol for API connections, data flows and semantic links
- ETH Grey for secondary modules and supporting infrastructure

Use accent colours sparingly.

Possible accent use:

- ETH Green for validated or successfully enriched data
- ETH Purple for advanced semantic or AI-enabled services
- ETH Bronze for selected categories or knowledge domains

Avoid ETH Red unless showing warnings, errors or critical states.

Lumina Core visuals should feel stable and trustworthy, not colourful or experimental.

## Relationship to Other Components

### Lumina Engine

Lumina Engine processes, enriches and computes.

Lumina Core provides shared data, services and access structures.

Visual relationship:

Engine may appear as a processing layer connected to Core through structured data flows and APIs.

---

### Lumina Studio

Lumina Studio manages, reviews and configures.

Lumina Core provides the underlying data and services that Studio interacts with.

Visual relationship:

Studio may appear as an operational dashboard connected to Core service modules.

---

### Lumina Experience

Lumina Experience provides user-facing discovery and interaction.

Lumina Core provides access to structured data, metadata and semantic services.

Visual relationship:

Experience may appear as a user interface layer drawing reliable knowledge services from Core.

## Avoid

Do not show Lumina Core as:

- a single database cylinder
- a server room
- a cloud icon cluster
- a black-box AI system
- a magical intelligence centre
- a cyberpunk control room
- a dense infrastructure diagram
- a chaotic network of services
- a purely technical backend without library meaning

Avoid visualising Core as invisible plumbing only.

It should be technical enough to communicate infrastructure, but meaningful enough to communicate knowledge services.

---

It should connect:

The image should communicate that Lumina Core is the trusted foundation that allows the rest of Lumina to work consistently.
